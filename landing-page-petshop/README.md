# Amora Pets - Landing Page

Este projeto é uma landing page funcional, responsiva e com aparência profissional para um pet shop fictício chamado **Amora Pets**. A marca foi pensada como um pet shop boutique, com foco em banho e tosa premium, atendimento humanizado, organização no agendamento foi baseada também na minha cachorra chamada "Amora".

## Objetivo do projeto

A página foi desenvolvida para consolidar conhecimentos de HTML, CSS, JavaScript e lógica básica de programação. Ela funciona como um cartão de visitas digital da marca, apresentando proposta de valor, serviços, diferenciais, imagens de apoio e um formulário de contato com validação básica no front-end.

## Identidade visual

A identidade da Amora Pets utiliza tons de creme, amora, coral, verde menta e azul escuro. A combinação passa acolhimento, cuidado e profissionalismo. A tipografia usa fontes externas do Google Fonts: **Baloo 2** para títulos e marca, criando uma personalidade amigável, e **Nunito** para textos, botões e formulários, garantindo boa leitura.

## Seções implementadas

A seção de destaque apresenta chamada comercial forte, botões de ação, selos de confiança e imagem principal do pet. Foram utilizados `position: relative` e `position: absolute` para criar a composição visual da foto sobre o fundo orgânico.

A seção sobre explica a proposta da marca com linguagem profissional e clara. Ela também apresenta indicadores fictícios, como quantidade de pets atendidos, anos de experiência e taxa de satisfação. A organização usa CSS Grid para separar texto e estatísticas.

A seção de serviços apresenta cards visuais com imagens, etiquetas, títulos e descrições. Os cards usam `aspect-ratio`, bordas arredondadas, sombras e organização consistente para reforçar a aparência de landing page profissional. Os serviços incluem banho premium, tosa estética, orientação pet e produtos selecionados.

A seção de experiência reforça os diferenciais do atendimento, explicando como a Amora Pets organiza o agendamento, recebe o pet e acompanha cada etapa. Também foi adicionada uma galeria com mais imagens para deixar a página mais visual e convincente.

A seção de contato possui dados comerciais fictícios e um formulário funcional na camada de front-end. O JavaScript valida nome, e-mail, telefone e mensagem. Se houver erro, o campo recebe destaque e uma mensagem orienta o usuário. Quando todos os campos estão corretos, a página exibe uma confirmação de envio.

## Tecnologias aplicadas

Foram utilizadas as três linguagens exigidas: HTML, CSS e JavaScript. O HTML usa tags semânticas como `header`, `nav`, `main`, `section`, `article`, `form` e `footer`. O CSS aplica variáveis, Flexbox, Grid, responsividade, pseudo-elementos, `aspect-ratio`, imagens responsivas e técnicas de posicionamento. O JavaScript manipula o DOM para controlar o menu mobile e validar o formulário sem recarregar a página.

## Responsividade

A página foi planejada para desktop, tablet e celular. Em telas menores, as colunas se reorganizam, o menu vira um botão expansível, a galeria se adapta para uma coluna e os cards mantêm legibilidade. Dessa forma, a landing page continua funcional e organizada em diferentes tamanhos de tela.
